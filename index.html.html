Understanding Quality Scores

  • Clarity Score: How well-defined your objective is
  • Specificity: Level of detail in requirements and metrics
  • Context: Technical and domain information provided
  • Completeness: Overall coverage of all enhancement fields

Pro Tips for Better Prompts

  • Be Specific: "User authentication" â†’ "JWT-based auth with refresh tokens"
  • Include Context: Mention your tech stack, team size, and project phase
  • Define Success: What does "good" look like? Include metrics
  • Add Examples: Show input/output samples when relevant
  • Specify Format: Code style, documentation format, file structure

Common Mistakes to Avoid

  • Being too vague: "Make it better" doesn't help
  • Conflicting requirements: Check for contradictions
  • Missing edge cases: Think about error scenarios
  • Ignoring constraints: Consider limitations upfront

The Science Behind Better Prompts

Our tool uses the C.R.A.F.T. methodology to transform vague prompts into comprehensive instructions that AI models can understand and execute effectively.

C - Context: Adds relevant background information R - Role: Defines expertise level and perspective A - Action: Clarifies specific tasks and steps F - Format: Specifies output structure and style T - Target: Optimizes for your chosen LLM

Step-by-Step Process

  1. Input Analysis: We parse your original prompt to identify key objectives
  2. Context Enrichment: Add technical specifications and domain knowledge
  3. Requirement Mapping: Structure your needs into clear, actionable items
  4. Quality Enhancement: Apply best practices for your target LLM
  5. Output Generation: Create a comprehensive prompt with all necessary details
